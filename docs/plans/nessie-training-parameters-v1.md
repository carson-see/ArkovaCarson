# Nessie Training Parameters v1 — 2026-04-15

> **Status:** authoritative parameters for all Nessie training going forward.
> **Supersedes:** the v6–v26 generalist sprawl. No training run starts unless it conforms to this doc.
> **Companion:** `gemini-training-parameters-v1.md` (separate model, separate purpose, no overlap).

## 1. What Nessie is, and is not

**Nessie is** a narrow, structured-extraction LoRA fine-tune of Llama 3.1 8B Instruct, hosted on RunPod via vLLM. One job: read PII-stripped credential text and emit a strict JSON object matching `ExtractedFieldsSchema`.

**Nessie is NOT:**
- A fraud-detection model (that's Gemini's job — see `gemini-training-parameters-v1.md`)
- A reasoning model (no chain-of-thought tokens)
- A multi-task model (one task: extract → JSON)
- A general-purpose chat model (refuse anything off-task)

This narrow scope is deliberate. The v6–v26 generalist approach failed because it tried to teach the same 8B model 12+ regulatory domains plus extraction plus fraud signals — none well.

## 2. Infrastructure (locked)

| Stage | Platform | Why |
|---|---|---|
| Fine-tune (SFT/LoRA) | **Together AI** | Cheapest LoRA SFT; ~$10–30 per 5K-example run |
| Merge LoRA + base | **RunPod (one-off pod)** | Has GPU, has bandwidth, no local disk hit |
| Push merged to HF | **RunPod (same pod)** | Direct push from pod, fast |
| Serve | **RunPod serverless vLLM** | Cost-effective serverless GPU, scales to zero |
| Eval | **Local CLI hits RunPod** | `npx tsx run-eval.ts --provider nessie --sample 50` |

**No local merge. No local upload. Ever.** The v26 attempt that filled local disk to 99% must not happen again.

## 3. Base model (locked)

```
meta-llama/Meta-Llama-3.1-8B-Instruct-Reference   (Together's hosted version)
```

Why 8B Instruct:
- 8B fits on a single A6000/L40S (~$0.40–0.60/hr serverless on RunPod)
- Instruct version follows JSON formatting instructions reliably out-of-box
- Reference variant is what Together's fine-tuning expects

**Never start a new fine-tune from an existing Nessie LoRA.** Each version starts fresh from Llama 3.1 8B Instruct base. Catastrophic forgetting from stacked LoRAs is what killed v6–v26.

## 4. LoRA hyperparameters (locked unless ablation justifies change)

| Param | Value | Rationale |
|---|---|---|
| `lora_r` | 16 | Standard for 8B extraction fine-tunes; doubles rank from default 8 for richer adapters |
| `lora_alpha` | 32 | 2× rank, standard ratio |
| `lora_dropout` | 0.05 | Light regularization |
| `target_modules` | `["q_proj","k_proj","v_proj","o_proj"]` | Attention-only LoRA — sufficient for output-format learning |
| `n_epochs` | 3 | More than 3 → overfit on small datasets; less → underfit format |
| `batch_size` | 8 | Together default for 8B |
| `learning_rate` | 1e-4 | LoRA standard; 5e-5 if loss is unstable |
| `warmup_ratio` | 0.05 | Standard |
| `weight_decay` | 0.01 | Standard |

**Variation rules:**
- Increasing `lora_r` requires an ablation: train at r=16 and r=32 on same 50-example test set, deploy whichever wins. Don't just pick larger.
- Changing `n_epochs` requires showing eval F1 vs epoch curve. If F1 plateaus by epoch 2, drop to 2 to save cost.

## 5. Dataset format (locked)

Training file: JSONL, one example per line. Together's chat-completions format:

```json
{
  "messages": [
    {
      "role": "system",
      "content": "<NESSIE_CONDENSED_PROMPT — see services/worker/src/ai/prompts/nessie-condensed.ts>"
    },
    {
      "role": "user",
      "content": "Extract credential metadata from the following PII-stripped text:\n\n<text here>"
    },
    {
      "role": "assistant",
      "content": "{\"credentialType\":\"DEGREE\",\"issuerName\":\"...\",\"issuedDate\":\"YYYY-MM-DD\",...,\"confidence\":0.95}"
    }
  ]
}
```

**Mandatory dataset rules:**
1. Every assistant message must be **valid JSON parseable by `ExtractedFieldsSchema.safeParse()`**. Build a script that validates every line before submission.
2. Confidence field must be a float in `[0,1]`, must reflect uncertainty (don't hand-label everything 0.95 — the model learns to be overconfident).
3. No reasoning trace tokens. No `<reasoning>...</reasoning>`. No commentary. JSON only.
4. Date format always `YYYY-MM-DD`. Other formats get rejected by validation.
5. **Hold out 20% as test set.** Track which examples are train vs test in `dataset.manifest.json`. Never train on test.

**Anti-pattern dataset (what v6–v26 did):**
- Auto-generated by scripts with no quality gate
- Mixed domains (extraction + reasoning + jurisdiction analysis)
- Inconsistent JSON formatting (some had trailing commas, some had unquoted keys)
- No held-out test set

## 6. Single-domain rollout (the new way)

Train one credential type at a time. Deploy. Verify. Move on.

| Wave | Domain | Why this order |
|---|---|---|
| 1 | DEGREE | Highest production volume; uniform format; ~700 entries available in golden |
| 2 | CLE | High volume; format slightly more complex (jurisdictions, credit hours) |
| 3 | LICENSE | Public-record heavy; benefits from jurisdiction context |
| 4 | PROFESSIONAL | Catch-all certifications; useful for general-purpose extraction |
| 5 | CERTIFICATE | Course/CEU certs |
| 6+ | (decided after wave 5 eval data) | Re-prioritize based on production traffic mix |

**Naming convention:** `arkova-nessie-<domain>-v<n>` (e.g., `arkova-nessie-degree-v1`, `arkova-nessie-cle-v1`). Each domain has its own version chain. No more `v26-africa` style multi-domain names.

## 7. Definition of Done (per domain)

A `arkova-nessie-<domain>-vN` model is production-deployable iff:

| Gate | Threshold | Measured by |
|---|---|---|
| F1 on held-out test set | ≥ 85% | `run-eval.ts --provider nessie --sample 20` filtered to that domain |
| Confidence Pearson r | ≥ 0.7 | Same eval, calibration section |
| ECE | ≤ 10% | Same eval, calibration section |
| JSON parse success rate | 100% | Logged from production canary |
| p95 latency (warm) | < 5000 ms | RunPod metrics over 24h |
| Cost per 1K extractions | < Gemini Flash equivalent | Cost tracker `services/worker/src/ai/cost-tracker.ts` |
| 7-day production canary | No circuit breaker trip | Worker logs |

**No hand-waving.** A domain is "Done" only when this table is filled in with green checks in the corresponding `docs/eval/eval-nessie-<domain>-v<n>-<date>.md`. Jira "Done" alone is not sufficient.

## 8. Eval workflow per training run

```bash
# After Together training completes:
cd services/worker
npx tsx scripts/build-domain-dataset.ts DEGREE  # produces nessie-degree-v1-train.jsonl + test.jsonl
# Submit to Together (separate runbook)
# Wait for completion
# Run RunPod-native deploy (separate runbook)
# Update worker .env NESSIE_MODEL
npx tsx src/ai/eval/run-eval.ts --provider nessie --sample 20 --output docs/eval/
# Compare against Gemini Flash on same entries:
npx tsx src/ai/eval/run-eval.ts --provider gemini --sample 20 --output docs/eval/
# Diff the two reports. Document which Nessie wins / loses on. File issues.
```

## 9. Cost guardrails

- **Per training run:** ≤ $50 on Together. If a single domain costs more, the dataset is too big — narrow it.
- **Per deploy:** ≤ $5 on RunPod (one-off merge pod).
- **Per eval:** ≤ $2 (50-sample run on warm endpoint).
- **Monthly serving:** ≤ $200 baseline (workersMin=0, cold start tolerated).

**If we exceed budget:** stop and reassess. The v6–v26 spend with zero deployed value cannot repeat.

## 10. What we are explicitly NOT training Nessie on

(These belong to Gemini — see `gemini-training-parameters-v1.md`)

- Fraud signal detection
- Multi-step compliance reasoning
- Cross-jurisdiction analysis (Nessie just extracts the jurisdiction field; reasoning happens in Gemini)
- Free-text Q&A over public records (RAG layer uses embeddings + Gemini synthesis)
- Document tampering detection
- Diploma mill identification (uses external blocklist + Gemini)

This separation is what makes both models trainable in isolation.

## 11. Living parameter changes

To change anything in this doc:
1. Run a 50-example A/B test (current params vs proposed change)
2. Document delta in `docs/eval/`
3. Update this doc with the new locked value
4. Bump the `v1` version to `v2` in the filename

No vibes-based tuning.
