## Critical (P0) — Must Fix Before Launch

### BUG-UAT-01: Mobile sidebar does not auto-collapse
- **Severity:** CRITICAL
- **Affected:** All authenticated pages at mobile viewport (375px)
- **Sprint:** 5
- **Steps to reproduce:**
  1. Open any authenticated page (Dashboard, My Records, etc.)
  2. Set viewport to 375px width (mobile)
- **Expected:** Sidebar hidden or collapsed by default, with hamburger toggle to open
- **Actual:** Sidebar renders fully expanded, covering ~65% of the viewport. Main content squeezed into ~120px strip on the right. App is unusable on mobile.
- **Root cause:** Sidebar does not detect mobile viewport on initial render. The "Collapse" button works when manually clicked, but sidebar defaults to expanded.
- **Affected files:** `src/components/layout/Sidebar.tsx`, `src/components/layout/AppShell.tsx`
- **Actions taken:** Identified during UAT 2026-03-15. No fix attempted yet.
- **Resolution:** OPEN
- **Regression test:** None yet — needs E2E test for mobile sidebar state

### BUG-UAT-02: Console auth errors on every page load
- **Severity:** CRITICAL (production optics)
- **Affected:** Every page, every load
- **Sprint:** 5
- **Steps to reproduce:**
  1. Load any page in the app
  2. Open browser console
- **Expected:** No errors in console
- **Actual:** `AuthApiError: missing destination name oauth_client_id in *models.Session` fires 6x on every page load
- **Root cause:** Supabase GoTrue session restoration attempts to read `oauth_client_id` field that doesn't exist. Likely Supabase SDK version mismatch or GoTrue server config issue.
- **Affected files:** `src/lib/supabase.ts` (client init), Supabase project GoTrue config
- **Actions taken:** Identified during UAT 2026-03-15.
- **Resolution:** OPEN — may require Supabase project settings update or SDK version alignment
- **Regression test:** None yet

### BUG-UAT-03: `/billing` route inaccessible — silent redirect to Dashboard
- **Severity:** CRITICAL
- **Affected:** Billing/Pricing page
- **Sprint:** 5
- **Steps to reproduce:**
  1. Log in as any user
  2. Navigate to `/billing`
- **Expected:** PricingPage renders with Free/Individual/Professional plan cards
- **Actual:** Silently redirects to Dashboard. No error, no feedback. No "Billing" link exists in sidebar navigation either.
- **Root cause:** RouteGuard or billing route config may have a guard condition that fails (e.g., checking for a billing-related profile field that's null). Sidebar doesn't include a billing nav item.
- **Affected files:** `src/App.tsx` (route config), `src/components/layout/Sidebar.tsx` (missing nav item), `src/pages/PricingPage.tsx`
- **Actions taken:** Confirmed route exists at `ROUTES.BILLING = '/billing'` in `src/lib/routes.ts`. PricingPage component exists. Route is wrapped in AuthGuard + RouteGuard.
- **Resolution:** OPEN
- **Regression test:** None yet
